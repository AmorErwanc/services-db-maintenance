# 架构决策记录（ADR）

## ADR-001 · 共享数据库维护脚本与业务项目解耦

**日期**：2026-07-16  
**状态**：已采纳

**上下文**：`llm_call_log` 是 dream-avatar、mingle 等业务共用的调用日志表。保留策略属于共享数据库治理，不应由任一业务项目私有脚本承担。

**决策**：建立独立的轻量维护项目；脚本运行在 `avatar-01` 宿主机，由单一 crontab 调度。原始调用记录统一保留 30 天，默认 dry-run，正式删除必须显式传入 `--apply`，每批 10,000 行。

**其他考虑过的方案**：放进 dream-agent 会制造错误归属；做成常驻服务或容器超出需求；最终均不采用。

**影响**：超过 30 天的所有业务调用原文都会被永久删除；表内空间通常只会变成可复用页，不承诺物理文件立即缩小。
